## 2026-04-03 - Python Code Injection via Shell Interpolation in CD Workflow
**Vulnerability:** The CD workflow used shell interpolation (`$VERSION`) inside a `python3 -c` script. An attacker controlling the `tag` input could provide a payload like `v1.0.0'); import os; os.system('...` to escape the string and execute arbitrary code.
**Learning:** Direct interpolation of untrusted shell variables into scripts (Python, JS, etc.) creates a high-risk injection vector. GitHub Action `inputs` and `client_payload` are untrusted and must be treated as such even if they are partially validated later.
**Prevention:** Map untrusted inputs to environment variables and access them via `os.environ` in Python, or use purpose-built tools like `jq --arg` for JSON manipulation which handle quoting and escaping safely. Always enforce strict execution mode (`set -euo pipefail`) in shell scripts.
