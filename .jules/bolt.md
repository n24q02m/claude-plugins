## 2024-05-18 - ls -A vs find for checking empty directories
**Learning:** Checking if a directory is not empty using `[ -n "$(find "$dir" -mindepth 1 2>/dev/null | head -n 1)" ]` instead of `[ "$(ls -A "$dir" 2>/dev/null)" ]` can theoretically avoid loading a massive directory into memory. However, `find` combined with a pipe and `head` is slower in typical small directories and requires GNU extensions (like `-mindepth 1`). Therefore, changing this established `ls -A` idiom without a proven bottleneck is an unnecessary micro-optimization that risks script portability.
**Action:** Do not replace `ls -A` with `find` for empty directory checks in simple bash scripts unless the directory is proven to contain tens of thousands of files and the `ls` command is blocking execution. Maintain readability and portability first.
